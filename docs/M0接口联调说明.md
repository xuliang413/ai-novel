# M0 接口联调说明

本文档用于本地验证 AI 小说 M0 后端骨架链路。

## 一、前置条件

- MySQL、Redis、Neo4j 已在本地 Docker 中启动。
- MySQL 使用当前 `dev` 配置连接 `saai` 库。
- Redis 使用当前 `dev` 配置连接 `127.0.0.1:6380`。
- Neo4j 使用 `bolt://127.0.0.1:8687`，账号密码为 `neo4j/your_password`。
- 本机 `1024` 端口如果被占用，可以临时改用 `11024` 启动后端。

## 二、启动后端

进入 `saai-be` 后启动。直接用 Maven 启动 `sa-admin` 时，先安装一次 `sa-base`，避免运行时拿到本地仓库里的旧依赖：

```powershell
$env:JAVA_HOME='D:\Develop\Java\ms-17.0.17'
$env:Path="$env:JAVA_HOME\bin;$env:Path"
mvn -pl sa-base install -DskipTests
mvn -pl sa-admin spring-boot:run "-Dspring-boot.run.arguments=--server.port=11024 --smart.job.enabled=false --localPath=D:/novel/ai-novel/runtime --novel.neo4j.uri=bolt://127.0.0.1:8687"
```

说明：

- `--server.port=11024` 用于避开本机已占用的 `1024` 端口。
- `--smart.job.enabled=false` 用于关闭 SmartJob 示例任务，避免联调时持续写入示例任务日志。
- `--localPath=D:/novel/ai-novel/runtime` 用于把 SmartAdmin 日志和 `ip2region.xdb` 写到项目本地目录，避免 Windows 下默认 `/home` 路径权限问题。
- `--novel.neo4j.uri=bolt://127.0.0.1:8687` 用于显式连接本地 Docker 中的新 Neo4j 端口。

## 四、登录获取 Token

除健康检查外，M0 业务接口需要登录态。

联调时可以使用 SmartAdmin 管理员账号 `admin`，并使用系统配置中的万能密码 `1024ok`。登录接口要求密码经过前端同款 SM4 加密，并携带验证码。

登录流程：

1. 请求 `GET /login/getCaptcha` 获取 `captchaUuid` 和开发环境返回的 `captchaText`。
2. 将万能密码 `1024ok` 按前端 `saai-web/src/lib/encrypt.js` 的 SM4 规则加密。
3. 请求 `POST /login` 获取 `data.token`。
4. 后续接口请求头携带 `Authorization: Bearer {token}`。

## 六、MySQL 验收 SQL

```sql
SELECT project_id, project_name, genre, protagonist, status, create_user_id
FROM t_novel_project
ORDER BY project_id DESC
LIMIT 1;

SELECT chapter_id, project_id, chapter_no, title, status, generation_session_id
FROM t_novel_chapter
ORDER BY chapter_id DESC
LIMIT 1;

SELECT session_id, project_id, chapter_id, chapter_no, provider, status, prompt_snapshot
FROM t_chapter_generation_session
ORDER BY session_id DESC
LIMIT 1;
```

## 七、Neo4j 验收 Cypher

```cypher
MATCH (p:Project {projectId: $projectId})
OPTIONAL MATCH (p)-[:HAS_CHARACTER {projectId: $projectId}]->(character:Character)
OPTIONAL MATCH (p)-[:HAS_LOCATION {projectId: $projectId}]->(location:Location)
OPTIONAL MATCH (p)-[:HAS_CLUE {projectId: $projectId}]->(clue:Clue)
OPTIONAL MATCH (p)-[:HAS_CHAPTER {projectId: $projectId}]->(chapter:Chapter)
RETURN p.name AS projectName,
       count(DISTINCT character) AS characterCount,
       count(DISTINCT location) AS locationCount,
       count(DISTINCT clue) AS clueCount,
       count(DISTINCT chapter) AS chapterCount;
```

约束检查：

```cypher
SHOW CONSTRAINTS
YIELD name
WHERE name IN [
  'novel_project_project_id',
  'novel_character_project_name',
  'novel_location_project_name',
  'novel_clue_project_name',
  'novel_chapter_project_number'
]
RETURN collect(name) AS constraintNames;
```

## 八、注意事项

- 使用 PowerShell 手写中文 JSON 时，请确保请求体按 UTF-8 字节发送，并设置 `Content-Type: application/json; charset=utf-8`。
- 枚举字段需要传枚举编码，例如 `MYSTERY`、`PROTAGONIST`、`BUILDING`、`MAIN`、`DORMANT`，不要传中文展示名。
- M0 的 `/novel/write/mock` 不调用真实大模型，只用于验证后端链路和数据闭环。
- Neo4j 写入遵循 `projectId` 隔离原则，`mysqlId` 只用于保存 MySQL 来源主键。
