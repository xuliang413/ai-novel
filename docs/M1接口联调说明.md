# M1 接口联调说明

本文档用于验证 AI 小说 M1 安全写作闭环的后端最小链路。

## 一、前置条件

- 已完成 M0 的 MySQL、Redis、Neo4j 和登录 Token 准备。
- 已执行 `saai-be/db/update.sql` 中的 M1 表结构增量：
  - `t_chapter_generation_session.intent_json`
  - `t_chapter_generation_session.content_review_json`
  - `t_chapter_generation_session.graph_patch_json`
  - `t_chapter_generation_session.inverse_patch_json`
  - `t_chapter_generation_session.operation_batch_id`
  - `t_graph_change_log`

## 三、M1 主链路

### 1. 启动写作并进入正文审阅

```http
POST http://127.0.0.1:11024/novel/write/start
Authorization: Bearer {token}
Content-Type: application/json; charset=utf-8
```

```json
{
  "projectId": 1,
  "chapterNo": 2,
  "pov": "林澈",
  "chapterGoal": "让林澈在旧钟楼发现逆行钟声的新证据"
}
```

期望结果：

- 返回 `sessionId`。
- 返回 `chapterIntent` 和 `contextPreview`。
- 章节状态为 `DRAFT`。
- 会话状态为 `CONTENT_REVIEW`。

### 2. 正文审阅通过并生成 GraphPatch

```http
POST http://127.0.0.1:11024/novel/write/content/pass
Authorization: Bearer {token}
Content-Type: application/json; charset=utf-8
```

```json
{
  "sessionId": 1,
  "chapterId": 1
}
```

期望结果：

- 返回 `graphPatch` 和 `inversePatch`。
- Patch 操作只包含后端白名单业务操作，例如：
  - `UPDATE_CHAPTER_SUMMARY`
  - `MARK_APPEARANCE`
  - `ADVANCE_CLUE`
- 章节状态为 `PENDING_GRAPH_CONFIRM`。
- 会话状态为 `PATCH_REVIEW`。

### 3. 确认 GraphPatch 并发布章节

```http
POST http://127.0.0.1:11024/novel/write/patch/confirm
Authorization: Bearer {token}
Content-Type: application/json; charset=utf-8
```

```json
{
  "sessionId": 1
}
```

说明：

- `operationIds` 为空时，执行 Patch 中 `selected = true` 的操作。
- 若要强制执行低置信操作，传入对应 `operationIds`。

期望结果：

- Neo4j 写入 Chapter 状态、出场关系和线索推进关系。
- `t_graph_change_log` 写入 `patch_json` 和 `inverse_patch_json`。
- 章节状态为 `PUBLISHED`。
- 会话状态为 `SUCCESS`。

### 4. 返回正文审阅

```http
POST http://127.0.0.1:11024/novel/write/patch/back
Authorization: Bearer {token}
Content-Type: application/json; charset=utf-8
```

```json
{
  "sessionId": 1
}
```

期望结果：

- 候选 GraphPatch 被丢弃。
- 正文保留。
- 章节状态回到 `DRAFT`。
- 会话状态回到 `CONTENT_REVIEW`。

### 5. 恢复写作状态

```http
POST http://127.0.0.1:11024/novel/write/recover
Authorization: Bearer {token}
Content-Type: application/json; charset=utf-8
```

```json
{
  "projectId": 1,
  "chapterNo": 2
}
```

期望结果：

- 恢复最近一次会话的 `chapterIntent`、`contextPreview`、`qualityCheck`。
- 若停在 `PATCH_REVIEW`，同时返回 `graphPatch` 和 `inversePatch`。

### 6. 撤销最近一次图谱变更

```http
POST http://127.0.0.1:11024/novel/write/undo
Authorization: Bearer {token}
Content-Type: application/json; charset=utf-8
```

```json
{
  "projectId": 1
}
```

期望结果：

- 执行最近一次 `APPLIED` 记录中的 `inversePatch`。
- `t_graph_change_log.status` 改为 `UNDONE`。
- 正文保留，章节状态改为 `PENDING_GRAPH_UPDATE`。

## 四、MySQL 验收 SQL

```sql
SELECT session_id, project_id, chapter_id, chapter_no, status,
       operation_batch_id,
       intent_json IS NOT NULL AS has_intent,
       graph_patch_json IS NOT NULL AS has_patch
FROM t_chapter_generation_session
ORDER BY session_id DESC
LIMIT 5;

SELECT chapter_id, project_id, chapter_no, title, status, generation_session_id
FROM t_novel_chapter
ORDER BY chapter_id DESC
LIMIT 5;

SELECT change_log_id, project_id, chapter_id, chapter_no,
       patch_id, operation_batch_id, status
FROM t_graph_change_log
ORDER BY change_log_id DESC
LIMIT 5;
```

## 五、Neo4j 验收 Cypher

```cypher
MATCH (chapter:Chapter {projectId: $projectId, number: $chapterNo})
OPTIONAL MATCH (entity)-[appearance:APPEARS_IN {projectId: $projectId, chapterNo: $chapterNo}]->(chapter)
OPTIONAL MATCH (chapter)-[advance:ADVANCES {projectId: $projectId, chapterNo: $chapterNo}]->(clue:Clue)
RETURN chapter.title AS title,
       chapter.status AS status,
       collect(DISTINCT entity.name) AS appearedEntities,
       collect(DISTINCT clue.name) AS advancedClues;
```
